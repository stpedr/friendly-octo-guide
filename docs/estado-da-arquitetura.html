Fluxo em camadas

Do cliente ao dado — cada bloco emite log, trace e métrica

A ordem é real: a requisição atravessa da borda até o dado, e a ingestão de linha entra em paralelo, direto da fábrica pro mesmo Kafka.

01 Cliente — PWA + Tauri core web

Painel admin e user, login com TOTP (RFC 6238), painel de linha ao vivo via SignalR. Um core web, três shells (Web/Desktop/Mobile).

PWA instalávelRUM sem PIIJWT curto + refresh rotativoWAF · ModSecurity/CRS

02 Gateway / BFF (YARP) implementado

Ponto único de entrada. Valida TOTP, emite JWT, RBAC/ABAC por planta/linha/turno, rate limiting distribuído e trace-id raiz (W3C).

Gateway.HostIdentityPlatform.AccessControlValkey · token bucket

03 Ingestão de linha (edge) implementado

Fábrica → nuvem, path próprio. Modbus/MQTT → buffer store-and-forward → Kafka. Quality gate valida faixa, drift e agora a confiança do relógio antes do dado virar verdade.

Edge.ProtocolGatewayTelemetry.Ingestquarentena p/ rejeitadachrony · NTP/PTP

04 Serviços de aplicação (.NET) implementado

Um processo por domínio, OpenTelemetry embutido, outbox pattern (evento na mesma transação do Postgres). Identity, Core.Execution, Notifications, Knowledge, Chatbot, Agents…

ASP.NET Minimal APIHotChocolate (GraphQL)Serilog + OTeloutbox → Kafka

05 Subsistema de IA (assíncrono) GPU: 1× A2000 Ada

Router/DLQ + workers por modelo, idempotência por job-id. LLM/visão/embeddings auto-hospedados — mas em 16 GB de VRAM é time-sharing com scale-to-zero, não um pod sempre-ligado por modelo.

Ai.Router.WorkerAi.Worker.Llm/Vision/EmbeddingsKEDA scale-to-zeroPredictive · Decision.Engine

06 Dados implementado

Valkey (cache), PostgreSQL (verdade + PITR + réplica), JSONB+pgvector via GraphQL, VictoriaMetrics (TSDB), MinIO WORM (lake) com linhagem OpenLineage. Consultável direto por DuckDB.

Data.ArchiverValkey · Postgres · MinIODuckDB sobre o lake

07 Espinha de observabilidade implementado

Todo log, trace e métrica converge aqui — nada sai do funil. OTel Collector → Loki (logs), Tempo (traces), VictoriaMetrics (métricas + FinOps via OpenCost), Grafana (painel unificado). Cliente nativo Tauri segue a mesma regra: um core, três shells.

OTel CollectorLoki · Tempo · VictoriaMetricsGrafana + Alertmanager + ntfySLO · Pyrra

Prática de engenharia

TDD é gate de CI, não cultura

Pipeline por bloco

  1. 00Gate TDD. PR que muda src/** sem tocar no *.Tests correspondente é reprovado.
  2. 01Build multi-stage, warnings = erros, nullable, cobertura mínima 80% no domínio.
  3. 02Testes unitário → contrato → SAST; edge contra simulador de linha, não hardware.
  4. 03Publish imagem assinada (cosign) + SBOM; o cluster recusa imagem não assinada.
  5. 04Deploy (GitOps) ArgoCD/Flux; canary + rollback via observabilidade.
